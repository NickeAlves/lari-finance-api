# Security Policy

## Supported versions

Only the latest version on the `main` branch receives security updates.

| Version       | Supported          |
| ------------- | ------------------ |
| main (latest) | :white_check_mark: |
| others        | :x:                |

## How to report a vulnerability

If you find a security vulnerability, please report it via:

- **GitHub Issues** — open an issue marked as confidential, or
- **Email** — nickalves88@gmail.com

Include in the message:
- Description of the problem and possible impact
- Steps to reproduce (endpoint, payload, etc.)
- Affected project version (commit hash, if possible)

You can expect a response within **7 business days**. Confirmed vulnerabilities will be fixed and communicated with a note in the commit history.

## Implemented security measures

| Area | Implementation |
|------|----------------|
| Authentication | JWT with HMAC-SHA; secret configurable via `JWT_SECRET` (minimum 32 bytes) |
| Passwords | BCrypt (Spring Security `BCryptPasswordEncoder`) |
| Session | Stateless — no cookies, no CSRF |
| CORS | Allowed origins configured via `CORS_ALLOWED_ORIGINS` |
| Data isolation | Each user accesses only their own records |
| SQL injection | Prevented via JPA/Hibernate with typed parameters |
| Migrations | Flyway with versioned DDL; Hibernate without auto-DDL (`validate`) |
| Secrets | All via environment variables; never in code or the repository |

## System scope

This project does **not** process payments, integrate financial gateways, or perform online billing. The API records manually submitted entries and calculates automatic financial allocations.

Sensitive data stored:
- User name and email
- BCrypt hash of the password (plain-text value is never persisted)

## Production recommendations

- Set `REGISTRATION_ENABLED=false` after creating the initial account.
- Set `SPRINGDOC_ENABLED=false` to hide the Swagger UI in production.
- Use `JWT_SECRET` with at least 32 randomly generated bytes.
- Ensure HTTPS — Railway manages TLS automatically; in other environments use a reverse proxy (nginx, Caddy, etc.).
